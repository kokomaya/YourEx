# YourEx 剧情重构 & i18n 文本分离设计书

## 1. 总体叙事框架

### 1.1 背景设定

你是星际探测飞船 **Meridian-7** 的最后一名系统工程师。

飞船在一次跃迁事故中坠入未知时空裂隙，引擎损毁，燃料耗尽，生命维持系统开始倒计时。你发出的所有标准频段求救信号都石沉大海——这片时空没有已知文明的痕迹。

直到第 47 个周期，探测器捕获了一段微弱的异常信号。它不来自任何已知协议，不属于人类编码体系。信号若隐若现，但有一个标记反复出现：**rEx**。

这不是攻击，不是噪声。这是一种 **语言**。

你必须破解这种语言，与这个未知文明取得联系，争取到他们的帮助——否则，Meridian-7 将永远漂浮在这片虚空中。

### 1.2 五幕剧情弧线

| 幕 | 章节 | 叙事主线 | 情感节奏 |
|---|---|---|---|
| 第一幕：接触 | Ch1 Signal Contact | 发现信号、尝试解析、初步建立联系 | 孤独 → 希望 |
| 第二幕：理解 | Ch2 Pattern Recognition | 信号结构变复杂，开始理解 rEx 编码规则 | 困惑 → 领悟 |
| 第三幕：对话 | Ch3 Syntax Awakening | 掌握语法，开始双向通信 | 挫折 → 突破 |
| 第四幕：协议 | Ch4 Transmission | 建立正式通信协议，传输求救信息 | 紧迫 → 信任 |
| 第五幕：回应 | Ch5 rEx | rEx 文明回应，终极解码，获得帮助 | 高潮 → 解脱 |
| 尾声 | Ch6 Origin | 外星零件协议适配 / 起飞回家 | 融合 → 启航 |

## 2. i18n 文本分离方案

### 2.1 设计原则

- 所有面向用户的文本（story、dialogue、feedback、UI 标签）提取到独立的 locale 文件
- 代码中只引用 key，不硬编码文本
- 默认语言：`zh-CN`，后续可扩展 `en`、`ja` 等

### 2.2 文件结构

```
src/
  i18n/
    index.ts              ← getText(key, locale?) 入口
    types.ts              ← TextKey 类型定义
    zh-CN/
      welcome.ts          ← 欢迎动画文本
      chapters.ts         ← 章节标题、intro、complete 台词
      levels.ts           ← 各关卡 story、promptChallenge、hints、feedback
      ui.ts               ← 按钮、标签等 UI 文本
      rewards.ts          ← 奖励动画文本
    en/                   ← 未来扩展
      ...
```

### 2.3 受影响的现有文件

| 文件 | 当前硬编码文本 | 改造方式 |
|---|---|---|
| `src/story/dialogues.ts` | welcome、chapterIntro、chapterComplete、rexSignals | 迁移到 `i18n/zh-CN/chapters.ts` 和 `welcome.ts` |
| `src/data/levels/*.json` | story、promptChallenge、hints、feedback | 迁移 story/feedback 到 `i18n/zh-CN/levels.ts`，JSON 只保留 id/input/expected 等结构数据 |
| `webview-ui/src/components/Welcome/index.tsx` | BOOT_LINES、STORY_LINES 等 | 改为从 `window.__YOUREX_LOCALE__` 注入或 message 传入 |
| `src/engine/rewardBuilder.ts` | CHAPTER_NAMES、CHAPTER_PREVIEW | 迁移到 `i18n/zh-CN/rewards.ts` |
| `src/ui/sidebar/missionMap/` | 内联 HTML 中的中文文本 | 通过 message 传入本地化文本 |

### 2.4 实现优先级

本设计书 **不做 i18n 实现**，仅确立结构。后续实施时按此方案迁移。
